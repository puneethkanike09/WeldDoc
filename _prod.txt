
Weld.Doc
Product Specification Document
Welder Qualification &amp; Compliance Management Platform
Version 1.0  |  June 2025


1. Product Overview
Weld.Doc is a SaaS platform for digitizing and automating welder qualification management across industrial organizations. It replaces manual, paper-based documentation workflows with a structured digital system that enforces compliance with major welding standards (ASME IX, ISO 9606, AWS D1.1), tracks qualification lifecycles, and generates verifiable certificates.

1.1 Problem Statement
Currently, managing 20 welders per month requires:
Minimum 32 hrs/month of manual documentation effort (~INR 6,500 in labor cost)
40-50 printed pages per month in color and B/W prints (~INR 500)
Manual Excel-based expiry tracking across 6-month, 2-year, and 4-year milestones
8 distinct document types per welder (Qualification Plan, Certificate, DT/NDT Reports, Supporting MTC, Validation Records, Welder Master List)
10 NDT/DT reports accumulated per welder over a 4-year tenure

Total estimated cost: INR 7,000/month for 20 welders — and this scales linearly with workforce size.

1.2 Solution Summary
Weld.Doc digitizes the entire workflow: data input, range-of-approval calculation, document generation, digital signing, QR code validation, and expiry alerts. The platform is built for quality managers, welding engineers, and compliance officers in industries that weld to pressure vessel, structural, or pipeline codes.

3. Functional Requirements
3.1 Welder Qualification Input (WPQ)
The system must accept structured input for each welder qualification event.

Required Input Fields
Welder personal details: name, ID/UID, date of birth, employer
Welding process (SMAW, GTAW, GMAW, FCAW, SAW, etc.)
Base material: P-number/M-number/S-number or material group per standard
Filler material: F-number, A-number, classification, batch/heat number
Shielding gas: type, mixture, flow rate (where applicable)
Joint configuration: groove type, position, backing, backing material
Test piece dimensions: diameter, thickness, wall thickness
Welding parameters: preheat, interpass temperature, PWHT
Date of qualification test
Examiner / Inspector details: name, stamp reference, certification body

WPQ Clone Feature
The system shall allow cloning of an existing WPQ record to create a new qualification with pre-filled data. This significantly reduces data entry time for repeat qualifications or similar test scenarios. Cloning must create a fully independent new record — changes to the clone must not affect the original.

Template Support
Input forms must support and render correctly under all three standards:
ASME Section IX — WPQ form layout with P-numbers, F-numbers, A-numbers
ISO 9606 — Material groups, test piece geometry, standard-specific range rules
AWS D1.1 — Structural code variables and essential variable tracking

3.2 Automatic Range-of-Approval Calculation
This is the core automation engine of Weld.Doc. On submission of qualification input data, the system must auto-calculate the full range of approval for each essential variable per the selected standard.

Feature
Description
Technical Notes
Base Material Range
Auto-map the qualified P-number/material group to the range of base materials covered by the qualification
Logic must follow ASME IX QW-423 / ISO 9606 Table 5 / AWS D1.1 Table 4.5 — implement as a rules engine or lookup table, not hardcoded
Filler Material Range
Calculate F-number and A-number coverage from the deposited filler
Store filler classification matrix; must support multi-process qualifications
Thickness Range
Calculate qualified thickness range (min/max) from test piece thickness
Different formulas per standard; store as parametric rules keyed by standard + variable
Diameter Range
Compute pipe diameter coverage from OD of test piece
Applies primarily to ASME IX and ISO 9606 pipe qualifications
Position Coverage
Expand qualified positions from the test position per standard rules
E.g., 6G covers all positions in ASME IX; must handle standard-specific position tables
Process Coverage
Track whether a multi-process qualification covers each process independently
Flag if a combination WPS is required vs. independent coverage

Range data must remain viewable at any time after creation — including when referencing a WPQ during Weld Map or Weld Log workflows.

3.3 Certificate Generation
The system must generate a single-page Welder Qualification Certificate (WPQ Certificate) upon completion of qualification. The certificate must:
Auto-populate all qualification data and calculated ranges
Embed examiner/inspector stamp image (uploaded once per organization) and digital signature
Generate a unique QR code linked to the welder&apos;s live certificate record
Be exportable as a PDF and printable to A4 / Letter size
Regenerate automatically when a validation event extends the welder&apos;s qualification (updated expiry date reflected)

QR Code Specification
Each certificate carries a unique, immutable QR code tied to the welder&apos;s UID
Scanning the QR code opens a public-facing (no login required) web page showing: certificate details, current active/inactive status, range of approval, expiry dates
QR codes must remain valid for the lifetime of the record — even if the certificate is renewed or updated

3.4 Qualification Continuity &amp; Expiry Tracking
Welder qualifications have time-based validity that must be actively tracked. The system must manage three expiry windows:

Expiry Window
Action Required
6 months
Continuity check — welder must have used the process within 6 months or re-qualify
2 years
Standard qualification validity — most ASME IX and AWS D1.1 qualifications expire unless revalidated
4 years
Extended validity period — requires documented continuity evidence at each interval

Alert Requirements
System must send automated notifications before expiry at configurable lead times (e.g., 30 days, 60 days before expiry)
Notification channels: email (required); in-app dashboard alert (required); SMS optional
Alerts must identify: welder name, UID, process, expiry type (6m/2yr/4yr), exact expiry date
Expired qualifications must be visually flagged in the master list and dashboard

3.5 Auto-Validation on Supporting Document Submission
When a welder submits evidence of continuity (a supporting document confirming they have performed welding using the qualified process within the validity period), the system must:
Accept the uploaded supporting document (PDF or image)
Validate it against the qualification record (process, date range, welder UID)
Extend the qualification validity automatically if validation passes
Regenerate the certificate with updated expiry dates
Log the validation event with timestamp, document reference, and validator identity

3.6 Master List &amp; Dashboard

Welder Master List
A single-page organization-level view showing all welders. Must be filterable and exportable (PDF, Excel). Required columns:
Welder name, UID
Qualified processes
Material groups / P-numbers qualified
Qualification date and expiry dates (6m, 2yr, 4yr)
Active / Expired / Expiring Soon status badge
QR code link

Dashboard Requirements
Summary counts: total active welders, expiring in 30 days, expiring in 90 days, expired
Category gap view: show which welding process/position/material group combinations have zero active qualified welders — critical for production planning
Range-of-approval lookup: select any WPQ and view the full calculated range without re-entering data
Filter by: process, standard, material group, expiry status, date range


4. Roadmap Modules (Way Forward)
The following modules are planned as extensions to Weld.Doc, forming a full weld management lifecycle platform.

4.1 Weld.Procedure
Module for creating and managing Welding Procedure Specifications (WPS) and Procedure Qualification Records (PQR).
Create WPS documents linked to qualified PQR records
Store essential and supplementary essential variables per process and standard
Link WPS to welder qualifications to verify coverage before production weld assignment
Generate standard-compliant WPS PDF output

4.2 Weld Map
Module for digitizing fabrication drawing weld maps without requiring CAD software.
Upload PDF drawing — system overlays a joint identification layer
User maps each joint to a WPS and welder assignment directly on the drawing
No AutoCAD or CAD plugin required — browser-based PDF annotation
Outputs: weld map report with joint-to-WPS-to-welder traceability
Validates that assigned welder holds a qualification covering the joint&apos;s process, material, position, and thickness

4.3 Weld Log
Module for in-process data capture and traceability of produced welds.
Smart data entry during welding: actual parameters captured per joint/pass
Identification tags: joint number, welder UID, date, shift, WPS reference
Validation of produced welds against WPS allowable ranges — flags out-of-range parameters
NDT result logging: link inspection results to specific weld joints
Full traceability chain: material heat number → joint → welder → WPS → NDT result


5. Non-Functional Requirements

5.1 Data &amp; Storage
All welder records must be retained for a minimum of 10 years (regulatory requirement for pressure vessel fabricators)
Audit trail: every record creation, edit, and validation event must be logged with user identity and timestamp — immutable
Document storage: PDFs of generated certificates and uploaded supporting documents must be stored in cloud object storage

5.2 Access Control
Role-based access: Admin, Welding Engineer, QA Inspector, Read-only Viewer
Organization-level data isolation (multi-tenant architecture)
QR code public pages require no authentication — all other features require login

5.3 Integrations
Email service: transactional email for expiry alerts and certificate delivery
Digital signature: either built-in (stamp image + name) or integration with e-sign provider
Export: all documents exportable to PDF; master list exportable to Excel/CSV

5.4 Performance
Range-of-approval calculation must complete in under 3 seconds for any single qualification
Certificate PDF generation must complete in under 10 seconds
Master list must load within 3 seconds for up to 500 welders

5.5 Compliance &amp; Security
Data at rest and in transit must be encrypted (AES-256 / TLS 1.2+)
Generated documents must include tamper-evident metadata (hash/checksum)
QR code validation page must be served over HTTPS

7. Glossary
Term
Definition
WPQ
Welder Performance Qualification — the formal record of a welder passing a qualification test
WPS
Welding Procedure Specification — the documented procedure a welder follows
PQR
Procedure Qualification Record — the test record that qualifies a WPS
DT
Destructive Testing — physical testing of weld samples (bend test, tensile, etc.)
NDT
Non-Destructive Testing — inspection without destroying the weld (RT, UT, PT, MT, VT)
MTC
Material Test Certificate — mill certificate verifying material properties
P-number
ASME IX grouping of base metals with similar weldability properties
F-number
ASME IX grouping of filler metals
A-number
ASME IX classification of deposited weld metal chemical composition
QR Code
2D barcode embedded in the certificate; links to live welder record
UID
Unique Identifier assigned to each welder in the system
Range of Approval
The full scope of variables (materials, thickness, position, etc.) a welder is qualified to weld under a given WPQ
Continuity
The requirement that a welder must have used a process within the validity window to maintain qualification
